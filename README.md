# 中山大学 LMS 视频自动播放器



一个用于 **中山大学在线教学平台（LMS）** 的 Userscript 脚本。  

它会在课程视频页面自动完成以下流程：



- 自动等待视频加载

- 自动开始播放

- 自动设置播放倍速

- 可选跳过已完成（100%）的视频

- 当前视频播放结束后自动跳转到下一个视频

- 遇到加载失败时自动重试

- 视频长时间无进度时自动跳过

- 在页面右下角显示当前运行状态


适合用来减少重复手动点播切换视频的操作。



---



## 功能特性



### 1. 自动播放视频

脚本会等待页面中的视频播放器加载完成，然后自动尝试开始播放。



### 2. 自动设置倍速

支持在配置中设置播放速度，例如：



- `1`：正常速度

- `1.5`：1.5 倍速

- `2`：2 倍速



> 实际可用倍速取决于 LMS 页面和浏览器策略。



### 3. 自动跳过已完成视频

当检测到当前视频进度已经达到 `100%` 时，脚本可直接跳到下一个视频，避免重复播放。



### 4. 自动跳转到下一个视频

当前视频结束后，脚本会自动寻找下一个视频地址并跳转。



优先使用：



- 页面中的视频选择下拉框

- 页面中的“下一个活动”链接



### 5. 异常恢复与重试

如果播放器迟迟没有加载出来，脚本会在一定次数内自动重试。



### 6. 卡住检测

脚本会定时检查视频播放进度：



- 若视频长时间没有推进

- 且持续达到设定阈值



则会判断该视频可能已卡住，并自动跳转到下一个视频。



### 7. 页面状态浮窗

脚本会在页面右下角显示状态提示，例如：



- 正在初始化

- 等待播放器加载

- 正在播放

- 缓冲中

- 已完成，准备跳转

- 所有视频播放完毕

## 安装方式

### 方案一：通过 Userscript 管理器安装

推荐使用任意支持 Userscript 的浏览器扩展，例如：

* Tampermonkey
* Violentmonkey

安装步骤：

1. 在浏览器中安装 Userscript 管理器扩展
2. 新建一个脚本
3. 将本项目中的脚本内容完整复制进去
4. 保存
5. 打开中山大学 LMS 的视频页面即可自动生效

### 方案二：直接导入脚本文件

如果你已经将脚本保存为 `.user.js` 文件，可以直接拖入 Userscript 管理器中安装。

## 配置说明

脚本顶部提供了一个配置区，可按需修改：

```javascript
const CONFIG = {
    playbackRate: 1.5,
    videoLoadTimeout: 15000,
    retryDelay: 5000,
    maxRetries: 3,
    nextPageDelay: 1500,
    skipCompleted: true,
};
```

### 参数说明

#### `playbackRate`

播放倍速。

示例：

```javascript
playbackRate: 1.5
```

#### `videoLoadTimeout`

等待视频元素加载的超时时间，单位毫秒。

示例：

```javascript
videoLoadTimeout: 15000
```

表示等待 15 秒后还没找到视频元素就视为超时。

#### `retryDelay`

视频加载失败后的重试等待时间，单位毫秒。

示例：

```javascript
retryDelay: 5000
```

#### `maxRetries`

最大重试次数。

示例：

```javascript
maxRetries: 3
```

#### `nextPageDelay`

当前视频播放结束后，延迟多久再跳转到下一个视频页面，单位毫秒。

示例：

```javascript
nextPageDelay: 1500
```

这个延迟可以避免页面切换过快。

#### `skipCompleted`

是否跳过已完成（进度 100%）的视频。

示例：

```javascript
skipCompleted: true
```

* `true`：跳过
* `false`：即使已完成也重新播放

## 工作原理

脚本大致按照以下顺序运行：

1. 页面加载完成后启动脚本
2. 检查当前视频是否已经完成
3. 如果已完成且允许跳过，则直接寻找下一个视频
4. 如果未完成，则等待视频元素出现
5. 尝试点击播放按钮并调用 `play()`
6. 设置播放倍速
7. 监听视频事件：

   * `playing`
   * `pause`
   * `waiting`
   * `error`
   * `ended`
8. 视频结束后自动跳到下一个视频
9. 若播放长时间无进度，则判定卡住并自动跳过

## 自动跳转逻辑

脚本会按以下优先级寻找“下一个视频”：

### 方式 A：读取页面中的视频选择下拉框

如果页面存在视频切换下拉框，脚本会找到当前项之后的下一个视频资源地址，并自动补充 `forceview=1` 参数。

### 方式 B：读取“下一个活动”链接

如果没有可用下拉框，则尝试读取页面中的“下一个活动”按钮链接。

### 找不到下一项时

如果当前已经是最后一个视频，脚本会提示：

```text
所有视频已播放完成
```

## 注意事项

1. **浏览器自动播放策略可能会影响自动播放**

   * 某些浏览器会限制未交互页面直接播放视频
   * 脚本已经同时尝试点击播放按钮和调用 `video.play()`
   * 若仍被拦截，通常在你进行一次页面点击后即可恢复

2. **倍速不一定始终生效**

   * 取决于播放器本身是否允许修改 `playbackRate`

3. **页面结构变化可能导致脚本失效**

   * 如果 LMS 页面 DOM 结构变更，选择器可能需要更新

4. **请合理使用**

   * 本脚本仅用于减少重复点击操作
   * 是否将观看行为计入课程要求，取决于平台自身规则与课程设置

## 常见问题

### 1. 脚本没有自动播放

可依次检查：

* 当前页面是否为视频资源页面
* Userscript 管理器是否启用
* 脚本匹配地址是否正确
* 浏览器是否拦截自动播放

也可以打开控制台查看日志输出。

---

### 2. 视频加载失败后没有恢复

可以尝试：

* 刷新页面
* 检查网络状态
* 适当增大 `videoLoadTimeout`
* 适当增加 `maxRetries`

---

### 3. 播放结束后没有跳到下一个视频

可能原因：

* 页面没有可识别的下一个视频地址
* 下拉框或“下一个活动”按钮结构发生变化
* 当前已经是最后一个视频

---

### 4. 视频被误判为卡住

如果你的网络较慢，建议：

* 延长检测间隔逻辑
* 或者修改相关判断条件，减少误跳过概率
